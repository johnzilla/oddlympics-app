<objective>
Harden the football-data.org `team.name` → `references/teams.json` slug mapping
so a label divergence (e.g. API "Korea Republic" vs catalog "South Korea",
"Bosnia-Herzegovina" vs "Bosnia and Herzegovina", "Czechia" vs "Czech
Republic", "Curaçao" diacritic bytes) can no longer silently leave
`teams.slug` NULL — which today means that team's confirmed subscribers get
ZERO kickoff emails, detectable only via a single `[ingest] no-slug` log line.

Purpose: This is the pre-launch silent-notification-loss launch risk recorded
in memory `notify-slug-mapping-launch-risk` and STATE.md Operator Next Steps.
The kickoff cron joins `teams t ON t.slug = COALESCE(ut.team_slug, v.team)`;
a NULL slug = silently dropped emails for that whole team. The fix replaces
the brittle exact-string match with a layered resolver (exact → normalized →
explicit alias map) shared by both ingest and backfill, adds a launch-gate
assertion that catches NULL slugs offline, and ships a deterministic offline
smoke proving all 48 slugs are reachable.

Output:
- `scripts/lib/resolve-team-slug.mjs` (new shared resolver)
- `scripts/smoke-team-resolver.mjs` (new offline coverage smoke)
- `scripts/ingest-schedule.mjs` + `scripts/backfill-team-slugs.mjs` rewired
- `scripts/launch-gate.mjs` new local-DB NULL-slug AC
- `package.json` smoke script entry
</objective>
