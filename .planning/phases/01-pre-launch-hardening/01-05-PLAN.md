- Directives are separated by `; ` (semicolon + space). The trailing semicolon is omitted (Caddy passes the value through verbatim; browsers tolerate either form).
- The `Content-Security-Policy-Report-Only` line is inserted between `X-Frame-Options "DENY"` and the `# Caddy adds Server: Caddy by default; hide it` comment.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → page render | CSP is the policy that constrains what the browser will execute/load on a page. Report-only mode does not block; it only reports. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / Elevation | XSS via injected inline script | mitigate (partial under report-only; full under enforce in plan 06) | The policy permits `'unsafe-inline'` for both script-src and style-src, so injected inline scripts ARE allowed in this phase (acknowledged accepted risk per D-17 / CONTEXT.md `<deferred>`). What the policy DOES block: external script origins not on the allow list (only `https://plausible.io` allowed), external connection origins (only `https://plausible.io` allowed), framing the site (`frame-ancestors 'none'` defeats clickjacking), arbitrary form-action targets, arbitrary base URIs, plugins (`object-src 'none'`). |
| T-05-02 | Information Disclosure | exfiltration via inline script connecting to attacker origin | mitigate | `connect-src 'self' https://plausible.io` blocks fetch/XHR/WebSocket to any other origin. Inline script can run, but cannot phone home. |
| T-05-03 | Spoofing | clickjacking via iframe embed | mitigate | `frame-ancestors 'none'` (plus existing `X-Frame-Options: DENY`) blocks all framing. |
| T-05-04 | Tampering | base-href hijack | mitigate | `base-uri 'self'` blocks `<base href="https://attacker.example.com">` injection. |
| T-05-05 | Information Disclosure | form-action redirect | mitigate | `form-action 'self'` keeps form submissions on-origin; an injected `<form action="https://attacker.example.com">` would be blocked by the browser. |
| T-05-06 | (Deployment) accidental enforcement before observation | n/a | mitigate | Header name is `Content-Security-Policy-Report-Only` (NOT `Content-Security-Policy`). Browsers report violations but do not block. Plan 06 is the deliberate flip after observation. |
| T-05-07 | Denial of Service | misconfigured CSP breaking the site | mitigate | The 1-2 day report-only window (Task 2) catches misconfigurations before they affect users. Acceptance criteria require zero browser-console violations on all four pages before approving plan 06. |
</threat_model>
