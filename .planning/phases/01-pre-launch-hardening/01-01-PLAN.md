## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `?status=` URL param | accept | Param is used as a key into a fixed object literal `COPY[status] || COPY.ok`. Unknown keys fall back to `ok`. No code execution path. |
| T-01-02 | Information Disclosure | rendered status copy | mitigate | `meta name="robots" content="noindex"` already present (preserved verbatim) — search engines do not index status pages. |
| T-01-03 | Cross-Site Scripting (S/T) | inline script + URL param | mitigate | All three injection sites use `.textContent` (not `.innerHTML`); CSP from plan 05 will further restrict inline script origins. The `c.cta` branch only toggles `hidden`, no string interpolation into HTML. |
