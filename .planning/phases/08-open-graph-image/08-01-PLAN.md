- `references/fonts/JetBrainsMono-Bold.ttf` — GitHub `JetBrains/JetBrainsMono` master, ~267KB, NameID 1 = `JetBrains Mono`, weight 700

RESEARCH §"Standard Stack" has the exact font URLs to document in the script's top-of-file comment.

The SVG `font-family` swap uses the unquoted bare strings `JetBrains Mono` and `Inter` (RESEARCH §"Anti-Patterns" notes resvg's SVG attr parser handles these unquoted; no problematic chars).

From `@resvg/resvg-js`:

The Resvg constructor takes (svgBuffer, options) where options.font has:
- `fontFiles: string[]` — absolute or cwd-relative paths to TTF/OTF files
- `loadSystemFonts: boolean` — set false for deterministic cross-platform render
- (optional `defaultFontFamily`, `serifFamily`, etc. — not used here)

options.fitTo has shape `{ mode: 'width' | 'height' | 'zoom' | 'original', value?: number }`
- Use `{ mode: 'width', value: 1200 }` — SVG viewBox `0 0 1200 630` is already 1200-wide, but specifying fitTo locks the output and makes the script robust against future SVG viewBox edits.

`.render()` returns `RenderedImage` with:
- `.asPng(): Buffer`
- `.width: number`
- `.height: number`

From `node:fs`:
- `readFileSync(path): Buffer` / `writeFileSync(path, data)`
- `statSync(path): { size: number, ... }`
- `existsSync(path): boolean`
- `Buffer.prototype.readUInt32BE(offset): number` — for PNG IHDR width/height parsing

From `node:child_process`:
- `execSync(cmd, { stdio: 'inherit' })` — for the LAND-02 shell grep
