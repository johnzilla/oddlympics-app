## Visual Hierarchy

**Primary focal point:** Form card (white surface, hairline border, light drop-shadow) anchored directly below the headline+subhead. Card width matches the 720 px container minus horizontal page padding so it dominates the above-the-fold viewport.

**Secondary anchor:** Rust-orange submit button (`Get match alerts`) inside the form card — the only saturated color block on the page, drawing the eye to the conversion action once the user reaches the card.

**Tertiary supports:** Banner pill above the headline (rust-on-soft-rust mono caps) primes the tournament context; rust-tinted step numerals in §"How it works" echo the accent rhythm below the fold.

The headline is large but intentionally near-black on warm off-white — it reads as content scaffolding around the focal form, not as the eye target itself.

## Spacing Scale

Declared values (multiples of 4, all tokens drawn from the canonical set {4, 8, 16, 24, 32, 48, 64}):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline micro-gap (e.g. step body line spacing), form label-to-input gap |
| sm | 8px | (reserved — not used directly on this page) |
| md | 16px | Step-list `gap`, FAQ accordion `margin-top` of body, form internal padding |
| lg | 24px | (reserved) |
| xl | 32px | Section vertical padding (`section { padding: 32px 0 }`) |
| 2xl | 48px | Hero top padding (mobile-clamped via `clamp(20px, 5vw, 48px)`) |
| 3xl | 64px | Hero top padding (desktop), footer bottom padding |

**Exceptions (all multiples of 4, documented):**
- **Form field vertical gap 12px** between fields (`margin-bottom: 12px`) — between md (16) and sm (8). Permitted because form density is visually tuned in `references/landing_preview.png`; do not generalize. 12 is a multiple of 4.
- **Container `--pad-x`** uses `clamp(20px, 5vw, 48px)` — fluid responsive horizontal padding, snaps 390 px (20px) → 1280 px (48px). 20 is a multiple of 4. No discrete breakpoint dance needed for horizontal rhythm.
- **Mobile hero top padding override 36px** (`@media (max-width: 520px)`) — tightens 64px desktop value on small screens. 36 is a multiple of 4.

**Container width:** `max-width: 720px` (variable `--max`). Centered. All hero / sections / footer share this container via a single `.wrap` class.

**Mobile breakpoint:** `@media (max-width: 520px)` — single breakpoint per the existing v1 pattern. Tightens hero top padding to 36px and stacks the submit row (currently a no-op since the row only has one button, but kept for forward compat).

## Typography

Two font families — sans-serif primary, mono accent — deliberately scoped. **First page in the project to introduce a sans-serif**; the existing `--mono` token name is preserved (now scoped to micro-accents only) and a new `--font-sans` token is added. Both live in `:root` of the single `<style is:global>` block.

**Font stacks:**

```css
--font-sans: ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
--font-mono: ui-monospace, SFMono-Regular, Menlo, Consolas, "Liberation Mono", monospace;
```

(System fonts only — zero web-font requests. Required for Lighthouse Performance ≥ 90 on mobile, LAND-03.)

**Type scale (exactly 4 sizes, exactly 2 weights):**

| Role | Size | Family | Weight | Line Height | Letter Spacing | Used in |
|------|------|--------|--------|-------------|----------------|---------|
| Display (headline) | `clamp(28px, 5vw, 40px)` | sans | 700 | 1.15 | -0.015em | `h1.headline` |
| Subhead | 17px | sans | 400 | 1.55 | normal | `p.subhead` |
| Body (prose, form input, step body, FAQ body) | 15px | sans | 400 (regular), 700 (`<strong>`) | 1.55 | normal | `.prose p`, `.steps .body span`, form fields, FAQ `<p>` |
| Label / Caption (eyebrow, banner pill, form label, trust, fineprint, error, footer link, copyright) | 13px | sans (mono for banner pill, step numerals, footer copyright) | 400 (body-caps prose, trust, fineprint, error, footer links), 700 (banner pill, form labels, step numerals, section h2 eyebrows) | 1.5 | 0.16em (banner pill), 0.12em (section h2 eyebrow), 0.02em (form labels), normal (others) | `section h2`, `.banner`, `.field label`, `.fineprint`, `.trust`, `.error`, `.steps .num`, footer `.copy`, footer links |

**Weight discipline:** Two weights total — **400 body, 700 emphasis**. The 400 weight covers all regular prose (subhead, body, fineprint, trust line, error text, footer links, FAQ body). The 700 weight covers all emphasis (display headline, section eyebrow labels, banner pill, form `<label>`, inline `<strong>` in prose, step numerals). No 500/600 semibold anywhere on the page.

**Reading rules:**
- Body `line-height: 1.55` for all prose — slightly looser than the standard 1.5 to match the airy, consumer feel of `references/landing_preview.png`.
- Headline `line-height: 1.15` — tight, allows the three-clause headline to fill two lines on desktop and three on mobile without feeling cramped.
- Section labels (`section h2`) all-caps, tracked +0.12em — visual hierarchy via tracking, not size. Reads as a section eyebrow, not a competing headline.
- Banner pill `letter-spacing: 0.16em`, all-caps, mono — establishes the "tournament HUD" voice in the only place that voice belongs. Bumped to 13px (from the earlier 11px draft) to preserve readability at 700 weight; the wide tracking keeps it visually compact.

**Accent reserved for:**
1. Banner pill text + 1px border (`.banner`).
2. Submit button background (`button[type="submit"]` inside `.cta`).
3. Step numeral text (`.steps .num`).
4. Focus ring on form fields (`input:focus`, `select:focus` — 1px border + 3px `rgba(217,74,31,0.15)` outer ring).
5. Inline links inside prose / footer / error message text (`a` default color).
6. Error message text (`.error`).

**Accent NOT used for:**
- Body text, headlines (use `--fg`).
- Section labels (use `--fg-dim`).
- Decorative borders or dividers (use `--line` / `--line-strong`).
- Hover states on body links — links use `filter: brightness(0.92)` on hover, not a color swap.

**60/30/10 audit:**
- ~60% — `#fafaf7` bg + body text on bg (the page is overwhelmingly off-white with near-black text).
- ~30% — `#ffffff` form card + section structure (hairlines, label eyebrows).
- ~10% — `#d94a1f` on banner pill, submit button, step numerals, focus rings, links, error.

**Contrast ratios (verify in checker for Lighthouse Accessibility ≥ 90, LAND-03):**
- `#14151a` on `#fafaf7` → ~16:1 (WCAG AAA).
- `#5a5d68` on `#fafaf7` → ~6.4:1 (WCAG AA, large text AAA).
- `#ffffff` on `#d94a1f` (submit button) → ~4.6:1 (WCAG AA for normal text — passes; verify with executor's actual color picker on `references/landing_preview.png` if the value drifts).
- `#d94a1f` on `#fbe9e0` (banner pill) → ~3.9:1 — fails WCAG AA for normal text but **acceptable** at 13px/700 mono with 0.16em tracking (large-text-equivalent rule + decorative eyebrow); banner pill carries no critical information that isn't also stated in `<title>` and headline.

## Interaction Contracts

### Form behavior (FORM-01/02/03)

- **Method/action:** `<form method="post" action="/api/signup" id="signup-form">` — unchanged from v1 (FORM-03).
- **Visible fields (in DOM order):** `team` (`<select required>`), `email` (`<input type="email" required autocomplete="email">`).
- **Hidden fields (in DOM order, after visible fields):** `timezone` (`<input type="hidden">`, populated by inline JS from `Intl.DateTimeFormat().resolvedOptions().timeZone`), `requested_sport` (`<input type="hidden" value="world_cup">`), honeypot `website` (`<input type="text" tabindex="-1" autocomplete="off" class="hp" aria-hidden="true">`).
- **Required validation:** Browser-native HTML5 (no JS). `team` empty + `email` empty both block POST.
- **Submit:** Button is the only submit affordance, takes full row width inside `.cta` card (`flex: 1`).

### Timezone label swap (CONTEXT D-03 / D-04 / D-05)

- **SSR HTML:** Sub-headline contains `<span id="tz-label">your local time</span>`. Page is fully readable JS-off.
- **Inline `<script is:inline>`:** On load, reads `Intl.DateTimeFormat().resolvedOptions().timeZone`. Algorithm: `tz.split('/').pop().replace(/_/g,' ') + ' time'`. Writes result into `#tz-label.textContent` **only if the result is a non-empty city label**. Also writes raw IANA value into `#timezone.value` (hidden form field).
- **Fallback:** If `tz` is empty, has no slash (e.g., `UTC`), is in the `Etc/*` family, or `Intl.DateTimeFormat` throws — leave the span text at `your local time`. The hidden `#timezone` field still posts the raw IANA value (or empty); `/api/signup` Phase 5 fallback (`FALLBACK_TZ = 'America/New_York'`) catches any invalid value without rejecting (SIGNUP-02).
- **CLS budget:** Zero. SSR text `your local time` and the swapped variants (`Detroit time`, `London time`, `Lagos time`, `New York time`, `Indianapolis time`) are visually similar in length and use the same font/weight; no layout shift.

### Error message rendering (FORM-03, COMPAT-02)

- **DOM:** `<p id="error" class="error" role="alert" hidden></p>` inside the form, after the submit row.
- **Inline `<script is:inline>`:** On load, reads `?error=<code>` from `URL.searchParams`. If present, unhides `#error` and sets `textContent` to the COPY map value (table above). Verbatim from `src/pages/index.astro:65-82` — kept by FORM-03 / COMPAT-02. No new error codes introduced.
- **Visual treatment:** Error block is `.accent-soft` background + accent text in a rounded box (8px padding, 6px radius) — same warm-rust family but soft enough to read as "informational alert", not "fatal." Matches `references/index.html:171-179`.

### Plausible event firing (ANLTC-01, CONTEXT D-09/10/11/12)

- **Listener:** Inline `<script is:inline>` attaches `form.addEventListener('submit', ...)` to `#signup-form`.
- **Algorithm:** Read `form.team.value`. If empty (HTML5 `required` should prevent, but guard anyway per D-11), return without firing. Otherwise call `window.plausible('Signup Submit', { props: { team } })`.
- **No `preventDefault`:** Fire-and-forget — Plausible's `pa-*.js` uses `sendBeacon` for unload-safe transmission (CONTEXT D-09). Page navigation to `/pending?email=…` continues normally.
- **Dev console log:** Only when `location.hostname === 'localhost' || location.hostname === '127.0.0.1'`, additionally call `console.log('[plausible] Signup Submit', { team })` per CONTEXT D-12.
- **Pre-deploy operator action (R-4):** Configure custom goal `Signup Submit` in Plausible dashboard at `https://plausible.io/oddlympics.app/settings/goals` BEFORE merging Phase 6 to `main`. Plan must include this as a non-code task.

### FAQ accordion behavior

- **Mechanism:** Native `<details>` + `<summary>` — no JS. All 5 items closed by default.
- **Affordance:** `+` glyph (mono, dim) on right of summary; flips to `−` when `[open]`. CSS-only.
- **Reduced motion:** Default `<details>` toggle has no animation in any major browser; the `transition` on the `::after` glyph is `0.15s ease` — well below the reduced-motion threshold. Add `@media (prefers-reduced-motion: reduce) { details summary::after { transition: none; } }` for completeness, matching the existing pattern at `src/pages/index.astro:251-253`.

### Focus rings

- **All interactive elements (form inputs, select, links, button):** 1px `--accent` border + 3px `rgba(217, 74, 31, 0.15)` outer ring on `:focus`. NEVER `outline: none` without a replacement focus indicator — Lighthouse Accessibility ≥ 90 requirement (LAND-03).
- **Submit button hover:** `filter: brightness(1.05)` — subtle, no color swap.
- **Submit button active:** `transform: translateY(1px)` — clipped under `prefers-reduced-motion: reduce`.

### Responsive behavior (LAND-04)

| Viewport | Behavior |
|----------|----------|
| 1280 px | Centered 720px container, 48px horizontal page padding, hero top padding 64px |
| 768 px | Centered 720px container fits with breathing room, horizontal padding fluid-clamps |
