<approach_notes>
**Inline token signing (~25 lines):**

```js
import { createHmac } from 'node:crypto';

const SECRET = process.env.MAGIC_LINK_SECRET || 'dev-only-insecure-secret-do-not-use-in-prod';

function b64url(buf) {
  return buf.toString('base64').replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, '');
}

function sign(data) {
  return b64url(createHmac('sha256', SECRET).update(data).digest());
}

function mintToken(email, opts = {}) {
  const ttlByPurpose = {
    confirm: 60 * 60 * 24,
    manage: 60 * 60 * 24,
    unsubscribe: 60 * 60 * 24 * 365,
    session: 60 * 60 * 24 * 30,
  };
  const ttl = opts.ttlSeconds ?? ttlByPurpose[opts.purpose ?? 'confirm'];
  const payload = { email, exp: Math.floor(Date.now() / 1000) + ttl };
  if (opts.purpose) payload.purpose = opts.purpose;
  const body = b64url(Buffer.from(JSON.stringify(payload)));
  return `${body}.${sign(body)}`;
}
```

This mirrors src/lib/token.ts mintToken exactly. Mint manage-purpose tokens for M2 (URL token first arrival), session-purpose tokens for M3–M6 (passed via Cookie header), and unsubscribe-purpose tokens for M8 (1y TTL).

**M9 DB-only path** (no HTTP):

Re-subscribe is verifiable without the magic-link round-trip: open a writable better-sqlite3 handle, INSERT/UPSERT a smoke row with confirmed_at set and unsubscribed_at set (simulating an unsubscribed user), then run the actual `markConfirmed` prepared statement (or its equivalent UPDATE SQL inline) and SELECT the row to confirm unsubscribed_at=NULL and confirmed_at is the current epoch second. Since markConfirmed is a server-side `db.prepare(...)` from src/lib/db.ts that runs at module load, the safest approach for the smoke is to issue the equivalent UPDATE directly via `better-sqlite3` rather than importing the .ts module. The UPDATE shape matches Plan 09-02's new markConfirmed body exactly.

The smoke is NOT testing whether the prepared statement is correctly registered (that's verified by `npm run build` in Plan 09-02); it's testing that the SQL behaviour matches D-07.
</approach_notes>
